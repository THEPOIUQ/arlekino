# Шаблоны коммитов и Pull Request’ов

Документ описывает согласованные правила ведения истории изменений в Shadow C2.

## Сообщения коммитов

### Шаблон
```
<type>: <краткое описание>

[необязательное подробное описание]

Refs: <ссылки на задачи или тикеты>
```

- `type` — одно из значений: `feat`, `fix`, `docs`, `refactor`, `test`, `chore`.
- Описание пишется в повелительном наклонении и не превышает 72 символов.
- Блок `Refs:` используется при наличии номера задачи или ссылки на обсуждение.

### Пример
```
feat: добавить статическую проверку синтаксиса

Запускаем py_compile для всех Python-файлов из скрипта run_static_checks.py,
чтобы отлавливать синтаксические ошибки до ревью.
```

### Настройка шаблона коммитов в Git
1. Скопируйте файл:
   ```bash
   cp docs/COMMIT_MESSAGE_TEMPLATE.txt .gitmessage
   ```
2. Укажите его в конфигурации:
   ```bash
   git config commit.template .gitmessage
   ```

После этого при выполнении `git commit` шаблон будет открываться автоматически.

## Pull Request’ы

### Шаблон
Репозиторий содержит файл `.github/pull_request_template.md`, который
автоматически подставляется при создании PR на GitHub. Не удаляйте секцию
«Quality checks»: она помогает ревьюерам быстро увидеть, какие команды запускались.

### Пример заполнения
```
## Summary
- описать ключевые изменения

## Testing
- [x] python3 scripts/run_static_checks.py
- [x] python3 test_suite.py
```

Перед отправкой убедитесь, что указанные проверки успешно выполнялись на локальной машине или в CI.
